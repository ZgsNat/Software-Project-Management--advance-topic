## 6. Công cụ sử dụng trong project

### 6.1 Jira

* Sử dụng: **Jira Software – Team-managed – Scrum**
* Jira dùng để:

  * Lập kế hoạch
  * Theo dõi tiến độ
  * Quản lý Sprint, Epic, User Story, Task

#### Cấu trúc trong Jira

* **Epic**: Nhóm chức năng lớn (feature chính)
* **User Story**: Mô tả yêu cầu theo chuẩn Agile
* **Task**: Chia nhỏ công việc để triển khai (design, code, test)

---

### 6.2 Confluence

* Confluence dùng để:

  * Lưu trữ tài liệu dự án
  * Mô tả chi tiết thiết kế, quyết định kỹ thuật
  * Làm documentation chính thức của project

#### Nguyên tắc bắt buộc

* Jira **không chứa mô tả chi tiết**
* Mỗi User Story / Task phải:

  * **Link tới một trang Confluence** để mô tả chi tiết

---

## 7. Tổ chức User Story và Task

* Mỗi User Story:

  * Được chia thành nhiều Task con
* Mỗi Task:

  * Có nội dung triển khai cụ thể
  * Có link sang trang Confluence tương ứng

### Nguyên tắc thiết kế

* Design và Implementation đi song song
* Documentation được cập nhật **theo tiến độ thực tế**

---

## 8. Sprint và quy trình triển khai

### 8.1 Số lượng Sprint

* Tổng cộng: **03 Sprint**

### 8.2 Nội dung từng Sprint

| Sprint   | Nội dung chính                                            |
| -------- | --------------------------------------------------------- |
| Sprint 0 | Initiation, Planning, Setup Jira & Confluence             |
| Sprint 1 | Phân tích yêu cầu, thiết kế tổng thể, triển khai phần đầu |
| Sprint 2 | Tiếp tục triển khai, hoàn thiện chức năng chính           |
| Sprint 3 | Hoàn thiện, kiểm tra, tổng hợp và chuẩn bị nộp            |

* Mỗi Sprint cần có **evidence rõ ràng trên Jira và Confluence**
